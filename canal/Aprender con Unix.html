Aprender con Unix

¿A que me refiero con aprender con Unix?

Con herramientas del estilo unix, puedes esperar a encontrarte a muchas patrones. Estos patrones se crearon como una comodidad y normalmente en los sistemas de tipo Unix (Gnu/Linux, OS X, *BSD) te podrás encontrar con estas ideas.

La bandera más importante

-h --help es la bandera con la que pudes pedir ayuda en cualquier herramienta de comandos.

Ejemplo:

  • cd --help
    • Este comando te da las opciones para entrar rápidamente a usar este comando.

Herramientas para leer documentación:

Introducción:

La mayoría de los sistemas de Unix tienen secciónes de sus manuales1, estas son las siguientes:

  1. Executable programs or shell commands
  2. System calls (functions provided by the kernel)
  3. Library calls (functions within program libraries)
  4. Special files (usually found in /dev)
  5. File formats and conventions, e.g. /etc/passwd
  6. Games
  7. Miscellaneous (including macro packages and conventions), e.g. man(7), groff(7)
  8. System administration commands (usually only for root)
  9. Kernel routines [Non standard]

apropos(1)

apropos(1) es una herramienta que busca en los manuales de las páginas y sus descripciónes.

Banderas Importantes:

  • -a --and hace un and lógico en donde tiene que encontrar dos términos especificados.
  • -r --regex usar una expresión regular en específico.

Ejemplos:

  • apropos vim
    • Busca todo lo que tenga vim en el nombre.
  • apropos emacs -a tags
    • Busca todo lo que tenga emacs y tags
  • apropos -r "direc*"
    • Imprime todo lo que tenga direc y algo más (es por eso que hay tantos resultados).

whatis(1)

Encuentra en una sóla línea lo que estas tratando de buscar. Se lleva muy bien con apropos(1).

Por ejemplo: Ya que sabes que existe una funcionalidad en una herramienta con apropos(1) puedes encontrar una breve descripción de lo que hace con whatis(1).

Banderas importantes

  • -r --regex expresiónes regulares

Ejemplos:

  • whatis emacs
    • Sólo retorna el resultado de la herramienta emacs y una breve descripción.

man(1)

Es la herramienta que normalmente se usa para encontrar la información de los manuales en los distintos sistemas operativos de tipo *nix.

Por ejemplo: man emacs da la información de la primera página del manual de emacs que encuentre.

Las partes que normalmente puedes encontrar en un manual son: NAME, SYNOPSIS, CONFIGURATION, DESCRIPTION, OPTIONS, EXIT STATUS, RETURN VALUE, ERRORS, ENVIRONMENT, FILES, VERSIONS, CONFORMING TO, NOTES, BUGS, EXAMPLE, AUTHORS, and SEE ALSO.1

Hay veces que el manual tiene varias páginas para un mismo comando, justo como es el caso para lo que es man(1) y man(7). Para acceder a estas páginas lo que uno puede hacer es poner el número de la página man 7 man o también man man(7).
