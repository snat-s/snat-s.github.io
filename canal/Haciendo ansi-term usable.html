Haciendo ansi-term usable

Introducción

Si has usado alguna vez emacs me imagino que haz tenido este problema.

Intentas usar ansi-term y no funciona porque:

  • No te deja copiar y pegar texto
  • El formato no funciona cuando instalas un paquete o tratas de usar una aplicación en la terminal (como htop).

Lo bueno es que esto se resuelve fácil, vamos a ver como quitarnos de estos dos problemas de encima.

Copiando y pegando texto

Tal vez no sepas esto pero es importante, los modos en Emacs mueven al mundo. El problema que principal se resuelve con una serie de comandos.

  • Para entrar a char-mode usas C-c C-j.
  • Para de cambiar de modo utilizas C-c C-k

Hacer que la terminal se vea bien

Podemos utilizar el paquete publicado dieggsy para hacer que la terminal se pueda usar y no tener problemas con el color cuando abrimos terminal con programas relativamente complejos.

Puedes agregar esta línea a tu .emacs. 

;; Inicializa package sources
(require 'package)
(setq package-archives '(("melpa" . "https://melpa.org/packages/")
                         ("elpa" . "https://elpa.gnu.org/packages/")))

;; Inicializa el package system

(package-initialize)
(unless package-archive-contents
  (package-refresh-contents))

;; Instala todos los paquetes si no están en la computadora
(require 'use-package)
(setq use-package-always-ensure t)

;; Instalas el paquete eterm
(use-package eterm-256color
  :hook (term-mode . eterm-256color-mode))
